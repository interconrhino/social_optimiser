# 06a — Recommendation: Formal Specification

**Session:** 0007
**Problem domain:** Food security / smallholder agriculture — retention on a climate-smart voice-advisory service
**Technique:** Restless Multi-Armed Bandit (RMAB) with Whittle-index policy, decision-focused learning for transition-model estimation, and a ProbFair-style per-subgroup minimum-pull-probability constraint for equity
**Algorithm:** **Whittle Index Policy** (Whittle 1988; Weber & Weiss 1990) on a **Collapsing / Partially-Observable RMAB** (Mate et al. 2020; Liu 2025), parameters learned via **Decision-Focused Learning** (Wang et al. 2023), with ProbFair equity floor (Li & Varakantham 2022)

---

## 1. Formal Problem Formulation

### Decision variables

- `a_{i,t} ∈ {0, 1}` (binary) — 1 iff farmer `i ∈ {1, …, N}` is scheduled for a personal follow-up call in month `t ∈ {1, …, T}`. `N ≈ 28,000`, `T ∈ {6, 7, 8}`.

### Objective function

Maximise the expected cumulative number of farmer-months of active engagement (farmer actually listens to the monthly automated voice message) across the pilot horizon:

```
max_{π}  E_π [ Σ_{t=1..T}  Σ_{i=1..N}  r(s_{i,t}, a_{i,t}) ]
```

where
- `π` is a non-anticipative policy mapping observation histories to action vectors,
- `s_{i,t} ∈ S_i` is farmer `i`'s latent engagement state in month `t` (discrete, small — e.g., {Highly engaged, Engaged, Drifting, Disengaged, Lost}),
- `r(s, a) = 1` if a farmer in state `s` and taking action `a` actually listens that month, else 0 (more generally, a state- and action-dependent listen probability).

### Constraints

**Budget (hard):** At most K = 400 calls per month.
```
Σ_{i=1..N} a_{i,t} ≤ K = 400       ∀ t ∈ {1, …, T}
```

**Binary assignment (hard):**
```
a_{i,t} ∈ {0, 1}    ∀ i, t
```

**Equity / fairness floor (hard side-constraint, via ProbFair):** For each subgroup `g` (e.g., (village, language) pair or coarser), the long-run expected pull probability per farmer in group `g` is at least `α · (K · T) / N` for a fairness floor α ∈ (0, 1]:
```
P[ a_{i,t} = 1 | i ∈ g ]  ≥  α · K / N    ∀ g, ∀ t
```
with `α = 0.5` as a reasonable starting floor (each subgroup gets at least half the population-average call rate). The user calibrates `α` based on district-auditor discussion.

**Cooldown (optional, soft):** No farmer is called in two consecutive months unless explicitly overridden:
```
a_{i,t} + a_{i,t+1} ≤ 1    ∀ i, t   (soft — enforced as a post-processing filter)
```

### Parameters

| Symbol | Meaning | Value / Source |
|---|---|---|
| `N` | Enrolled farmer population | ≈ 28,000 (from enrolment system, monthly refresh) |
| `K` | Monthly proactive-call budget | 400 (advisor throughput constraint) |
| `T` | Pilot horizon | 6–8 months (board-approved) |
| `S_i` | Per-farmer latent state space | Discrete, 5–10 states, learned from listen-pattern features. Recommended definition: listen-history sliding-window summary (e.g., # listens in last 3 months × recency of last listen × duration bucket). |
| `P^a_i(s' | s)` | Transition kernel for farmer `i` in state `s` under action `a ∈ {0, 1}` | **Estimated via decision-focused learning** from 5 years of monthly listen + call history plus static covariates. |
| `O(o | s)` | Observation model: probability of observing listen outcome `o` given latent state `s` | Estimated jointly with transitions. |
| `ρ_i` | Static covariates (district, village, language, farm size, crop mix, phone type, distance, collective membership) | **Known**, used as conditioning input to the transition model. |
| `ξ_t` | Seasonal context (district rainfall, market price) | **Known** month-by-month; used as a time-varying covariate. |
| `α` | Fairness floor (ProbFair) | Starting value 0.5; tunable. |

### Uncertainty structure

**Partial observability:** Each farmer's latent engagement state `s_{i,t}` is not directly observed; only the monthly listen outcome (a noisy function of the state) is observed. The policy maintains a **belief** `b_{i,t}(s)` — a probability distribution over `s` — updated after each observation via Bayes' rule against the observation model `O`.

**Restless transitions:** The latent state evolves even when the farmer is *not* called (passive action), not just when called. This is what makes the problem *restless* rather than a classical MAB.

**Confounded historical data:** Historical action assignment was generated by the rotation rule + helpline, not randomised. Two-stage MLE on this data identifies the wrong transition parameters; **decision-focused learning** fits parameters to minimise downstream policy regret, and a **pilot control arm** (rotation on part of the farmer base) gives unconfounded data going forward.

## 2. Recommended Algorithm

**Algorithm:** Whittle Index Policy on a Partially Observable / Collapsing RMAB, learned via Decision-Focused Learning, with ProbFair equity floor.
**Class:** Index-based policy for a Partially Observable Markov Decision Process (POMDP) approximated via Lagrangian relaxation of the budget constraint.

### Why this algorithm

1. **Structural fit.** The problem is the textbook finite-horizon, budget-constrained, partially-observable RMAB. Whittle's relaxation decomposes the N-arm joint problem into N independent single-arm belief-MDPs, solvable per farmer, then recombined via a sort by the per-farmer index at decision time.
2. **Real-world precedent.** Deployed by ARMMAN (maternal mHealth in India) on a near-identical operational pattern — weekly automated voice advisory, small human-caller budget for re-engagement, observational historical data, equity constraints. RCT-validated improvement over round-robin (Mate et al. 2022).
3. **DFL addresses the confounded-data problem** the user flagged (historical calls were not randomised).
4. **ProbFair gives a one-line equity guarantee** matching the user's "don't cut off villages/languages" requirement.
5. **Scale is easy.** 28,000 arms, 6–8 rounds, small state space → seconds per monthly run on a laptop.

### Theoretical properties

- **Optimality:** Whittle-index is asymptotically optimal as N → ∞ under indexability (Weber & Weiss 1990). Finite-N gap to the LP upper bound is empirically small (<5% in comparable public-health deployments).
- **Complexity:**
  - Fitting the transition model via DFL: minutes–hours on a laptop.
  - Per-month index computation: O(N · |S|³) ≈ 3 × 10⁷ ops (seconds).
  - Per-month selection: O(N log N) sort plus per-subgroup floor check.
- **Underlying problem is PSPACE-hard** (Papadimitriou & Tsitsiklis 1999); an exact solve is infeasible at N = 28,000, which is precisely why the index policy is the principled tractable choice.
- **Relaxed indexability** (Liu 2025) gives a verifiable online sufficient condition for the index to be well-defined in the partially observable case, with a bounded error guarantee.

### Key reference

Mate, A., Madaan, L., Taneja, A., Madhiwalla, N., Shah, S., Mehta, J., Hegde, A., Jain, M., & Tambe, M. (2022). *Field Study in Deploying Restless Multi-Armed Bandits: Assisting Non-Profits in Improving Maternal and Child Health.* AAAI 2022. (See also Whittle 1988; Mate et al. NeurIPS 2020; Wang et al. AAAI 2023; Li & Varakantham UAI 2022.)

### Known limitations

1. **Indexability must be verified** once the transition model is learned (use Liu 2025's online check).
2. **Finite-horizon regime (T = 6–8) is short** relative to asymptotic results; finite-horizon variants (Ghosh et al., AAMAS 2025) can be swapped in if the pilot suggests this matters.
3. **Identifiability from observational data is fundamentally limited**; the pilot's randomised control arm is necessary to close this gap.
4. **Model explainability is partial** — a post-hoc rationaliser is needed for field auditors.
5. **Arm independence is assumed** (no peer spillover through calls); this holds in the current programme design.

## 3. Implementation Guidance

### Data requirements

| Data item | Description | Format | Source / how to produce |
|---|---|---|---|
| Monthly listen outcome | Binary (listened ≥ threshold) + duration per farmer-month | Long-format table, ~28,000 × 60 rows | 5 years of telecom engagement logs |
| Historical call indicator | Whether a personal advisor call was placed per farmer-month | Long-format table, same key | Advisor call logs |
| Call outcome notes | Was the farmer reached? Short tone label. | Structured categorical | Advisor logs |
| Farmer attributes | District, village, language, age band, farm size, crop mix, education, phone type, distance to office, collective membership | Wide-format, one row per farmer | Enrolment system |
| Seasonal context | Monthly district rainfall and market prices | Panel by (district, month) | Public sources + internal procurement data |
| State representation | Derived feature: e.g., (listens-in-last-3-months, months-since-last-listen, rolling-listen-duration-bucket) | Derived monthly | Computed by the analysts as part of modelling |

### Key parameters to calibrate

| Parameter | What it controls | Starting value | Sensitivity |
|---|---|---|---|
| Number of latent states `|S|` | Granularity of the engagement-state model | 5–8 (e.g., Highly engaged / Engaged / Drifting / Disengaged / Lost) | **High** — re-check on pilot data |
| Fairness floor `α` (ProbFair) | Minimum per-subgroup call rate | 0.5 | **Medium** — affects equity/efficiency trade-off directly, should be discussed with field director |
| Cooldown length | Minimum months between calls to the same farmer | 1 month (no back-to-back) | **Low–Medium** — mostly an operational preference |
| DFL learning rate / regularisation | Transition model fitting | Standard Adam defaults + L2 on transition logits | **Low** given pre-existing ARMMAN code as reference |
| Subgroup definition `g` | Granularity of fairness floor | (district × language) — ~12 groups | **Medium** — finer subgroups tighten fairness but may over-constrain the selection |

### Algorithmic steps (monthly pipeline)

1. **Data refresh (Day 1 of month t):** Pull the latest listen outcomes and call logs through month t-1 into the analyst workspace.
2. **Belief update:** For each farmer, update the belief distribution `b_{i,t}` over their latent engagement state using the learned observation model and the latest listen outcome.
3. **Index computation:** For each farmer, compute the Whittle index `W_i(b_{i,t})` — the subsidy at which the farmer would be indifferent between being called and not called in the current belief. Use the code from Mate et al. 2020 / Liu 2025 as the reference implementation.
4. **Apply ProbFair floor:** For each subgroup `g`, ensure at least `⌈α · K · |g| / N⌉` farmers are selected from that subgroup — lift the lowest-ranked required farmers per group into the selection if needed.
5. **Sort and pick top-K:** After the floor lifts, sort remaining farmers by Whittle index and fill up to 400.
6. **Cooldown filter:** Remove any farmer called in month t-1 (replace with next in rank).
7. **Artefact:** Output a CSV/PDF listing the 400 selected farmers with per-farmer fields (name, district, advisor, last-call-date, index value, short rationale string).
8. **Handover to advisors:** Advisors call in their own prioritisation over the week.
9. **At end of month t:** Observations feed back in at step 1 of month t+1.

### Computational considerations

- Expected complexity at this scale: well under a minute total per monthly run on a laptop.
- Scaling: linear in N. Even at 100,000+ farmers the same pipeline runs in under a few minutes.
- If scale grows to 1M+ and state space grows: switch to neural index policies (e.g., arXiv:2510.22069, 2025) that learn a function approximation of the Whittle index. Not needed at 28k.

### Software recommendation

- **Language:** Python 3.11+.
- **Model fitting (DFL):** PyTorch; build on the reference implementation from the Harvard Teamcore RMAB codebase (public).
- **Index computation:** Adapt Mate et al. 2020 "Collapsing Bandits" reference code; the core routine is a per-arm finite-horizon dynamic program that is a few hundred lines.
- **ProbFair wrapper:** Li & Varakantham 2022 reference implementation.
- **Data plumbing:** pandas + SQLAlchemy for the log extracts; parquet for intermediate state.
- **Pilot randomisation & analysis:** R or Python (statsmodels) for the difference-in-differences or clustered-bootstrap analysis at pilot end.

## 4. Validation

### Evaluation metric

**Primary:** Total farmer-months of active engagement (= number of (i, t) pairs with `Listen_{i,t} = 1`) over the pilot window, on the **treatment arm**, vs the same quantity on the **rotation control arm**, normalised by arm size.

**Secondary:**
- Per-subgroup engagement rate (to confirm the equity constraint held).
- Distribution of inter-call gaps (to confirm no farmer was over-called or starved).
- Calls-to-reactivation ratio: among farmers who were drifting and got a call, fraction who listened the following month.

### Baseline to beat

**Current policy:** The rotation rule — 400 farmers/month picked in enrolment-date order, plus helpline overflow. This is already the planned control arm in the pilot.

**Expected improvement:** Based on the ARMMAN AAAI 2022 field study (the closest analogue), one can reasonably hope for a 10–30% lift in cumulative engagement over a 6–8 month window at the same call budget. The pilot estimates the realised lift for GreenFurrow specifically.

### Sensitivity analysis

Before locking the policy, vary and report:
1. **State space size `|S|`** (±2 states). Does the monthly selection list change materially? If yes, the chosen discretisation is sensitive and needs to be pinned down more carefully.
2. **Fairness floor `α`** (0.3, 0.5, 0.7). How much total engagement does the organisation sacrifice per 10 percentage points of α? This is the field director's conversation with the board.
3. **Cooldown** (0, 1, 2 months). Does enforcing a longer cooldown meaningfully redistribute calls to farmers who hadn't been called recently?
4. **DFL vs two-stage MLE.** On held-out months, does DFL-fit deliver better off-policy-evaluated engagement than MLE? If the gap is small, two-stage MLE is a simpler option for field use.

### Pilot approach

The user has board approval for a side-by-side pilot. Concrete design:

1. **Stratified randomisation of farmers into two arms** (treatment: Whittle + ProbFair; control: current rotation). Strata = (district × language group × recent-listen-band) to ensure balance on the features most predictive of engagement.
2. **Arm sizes:** ~14,000 each, so that detectable effect size at 6–8 months is around 2–3 percentage points of monthly engagement rate with 80% power.
3. **Both arms share the 400 call budget proportionally** (e.g., 200 calls/month to the treatment half via Whittle, 200 to the control half via rotation) to keep the comparison at the same per-farmer call rate.
4. **Monthly monitoring** of equity indicators to catch any subgroup that is being under-called, with a pre-committed rule for pausing/stopping the pilot if equity breaches appear.
5. **End-of-pilot analysis:** difference-in-differences on farmer-month engagement, clustered bootstrap over villages for inference. Report lift, confidence interval, per-subgroup breakdown, and subgroup-by-subgroup equity check.

### What will be produced each month during the pilot

- A **call list CSV** (400 farmers, with rationale string per farmer).
- A **dashboard** showing month-over-month engagement in each arm, per-subgroup engagement, and equity indicators.
- A **change log** noting any overrides the field director applied.
